CASE STUDY 03

Transforming Enterprise Benchmarking Data into Strategic Insights

How I redesigned The Hackett Group's finance dashboard to help senior executives compare 15,000+ metrics efficiently and make data-driven strategic decisions


Role

Lead UX/UI Designer

Timeline

7 months

Team

10 members (PM, Developers, Data Team)

Tools

Figma, InVision, Microsoft Clarity, Whimsical

About The Hackett Group

The Hackett Group is an enterprise benchmarking platform that measures performance excellence, accelerates business transformation, and uncovers breakthrough business insights for Fortune 500 companies. The platform processes over 15,000 financial and operational metrics to help organizations compare their performance against industry peers and best-in-class performers.

The Problem

Business Challenge

Senior finance executives were drowning in data but starving for insights. Despite having access to comprehensive benchmarking data across 15,000+ metrics, users couldn't efficiently compare performance indicators, switch between metric panels, or identify actionable insights within reasonable timeframes.

Key Issues Identified:

  • No efficient way to compare multiple metrics simultaneously
  • Slow navigation between different metric panels disrupted analysis flow
  • Senior team members lacked time to explore complex interfaces
  • Data overload prevented users from identifying strategic opportunities
  • No clear visual hierarchy to prioritize critical performance indicators
15,000+ metrics needed to be accessible and comparable

Research & Discovery

Working within an agile environment at Endava for The Hackett Group client, I conducted comprehensive user research to understand the complex needs of finance executives and data collectors.

1. Empathize Phase

Conducted stakeholder interviews with CTO and CEO, followed by extensive user interviews with finance team members and data collectors to understand their workflows, pain points, and time constraints.

Key Insights from Research

Time-Constrained Users

Senior finance team members had limited time to explore interfaces—they needed immediate access to critical insights without extensive navigation.

Comparative Analysis Need

Users needed to compare multiple metrics side-by-side simultaneously to identify patterns and make strategic decisions.

Complex User Roles

Different user types (Program Managers, PTE Managers, Data Collectors, Stakeholders) required different levels of access and functionality.

Data Collector Workflow

Internal data collection process was disconnected from end-user display, creating inconsistencies and delays in metric updates.

Missing Context

Raw metrics without context or benchmarks provided little actionable value for decision-making.

Excellence Quadrant Focus

Users specifically needed to identify metrics contributing to the "excellence quadrant"—The Hackett Group's key differentiator.

2. Define Phase

Created user journeys and user flows mapping core functionalities. Developed user personas for different roles (Data Collectors, Finance Managers, Executives) to understand how each would complete tasks and what information they needed along the way.

Core Problem Statement

How might we help senior finance executives efficiently navigate, compare, and extract strategic insights from 15,000+ benchmarking metrics without overwhelming them with data complexity?

The Design Process

3. Ideate & Prototype

Facilitated collaborative sessions with the 10-member team using Whimsical boards to create flowcharts and user stories. Rapidly iterated on concepts through sketching, wireframing, and functional mockups in Figma.

Solution 1: Multi-Panel Comparison Dashboard

The Challenge: Users needed to compare multiple metrics simultaneously but the existing interface only allowed viewing one metric panel at a time.

The Solution: Designed a flexible grid-based dashboard that allowed users to display multiple metric panels side-by-side. Implemented quick-switch functionality enabling users to toggle between different metric views without losing context or position.

Grid-Based Layout

Flexible panel system allowing 2-4 metrics displayed simultaneously for direct comparison

Quick-Switch Navigation

One-click metric switching without page reloads or lost context

Customizable Views

Users could save and recall preferred metric combinations for recurring analysis

Solution 2: Visual Data Hierarchy & Excellence Indicators

The Challenge: With 15,000+ metrics, users struggled to identify which data points were most critical for strategic decisions.

The Solution: Implemented a clear visual hierarchy using color-coding and excellence indicators. Designed a scoring system that highlighted metrics contributing to the "excellence quadrant"—helping users focus on high-impact performance areas.

  • Color-coded performance tiers (Excellent, Good, Average, Below Average)
  • Excellence quadrant badges highlighting top-performing metrics
  • Comparative advantage indicators showing where organizations excel vs. peers
  • Trend arrows and sparklines for quick pattern recognition

Solution 3: Contextual Benchmarking Display

The Challenge: Raw metrics without peer comparison context provided limited actionable insight.

The Solution: Integrated industry benchmarking directly into each metric panel, showing where the organization stood relative to peers, industry averages, and best-in-class performers.

Solution 4: Guided Onboarding & Presentation Mode

The Discovery: During development, we realized new users needed guidance to understand the platform's extensive capabilities.

The Solution: Designed a comprehensive onboarding tour and added a presentation mode feature (my suggestion, approved after team discussion). This allowed users to create slide-based reports directly from dashboard data for executive presentations.

4. Test & Iterate

Conducted A/B testing and usability studies throughout development. Used dynamic mockups in consecutive client meetings to explain, discuss, test, and rethink design decisions. Ensured deliverables matched client needs within the proposed timeframe.

Scope & Constraints Management

Original Timeline: 4-6 months for core functionality

Extended Timeline: 7 months after adding presentation functionality (2-week addition that significantly increased value)

Key Constraint: Designing for multiple user roles (Data Collectors, Program Managers, PTE Managers, Stakeholders) each with different needs and technical proficiency levels.

Qualitative Improvements

  • Enhanced decision-making speed: Executives could now identify strategic opportunities in real-time during meetings
  • Improved data democratization: Multiple user roles could access relevant information without overwhelming complexity
  • Excellence quadrant visibility: Clear visual indicators helped organizations focus on high-impact performance areas
  • Presentation capabilities: Built-in slide generation eliminated manual report creation, saving hours per analysis
  • Scalable design system: Flexible component architecture allowed easy addition of new metric types

Key Learnings

This project presented unique challenges in designing for data-intensive enterprise platforms with diverse user needs and massive scale requirements.

What I Learned

  • Context is king in data visualization: Raw metrics without benchmarking context provided minimal value—users needed comparative frameworks to understand performance
  • User role diversity requires flexible architecture: Different personas (Data Collectors, Managers, Executives) needed different information density and interaction patterns within the same platform
  • Collaborative iteration accelerates alignment: Figma's collaborative features enabled real-time feedback loops with developers and product owners, significantly reducing miscommunication
  • Feature discovery happens during development: The onboarding tour and presentation mode weren't in the original scope but emerged as critical needs through user testing
  • Agile methodology enables adaptive design: Working in an agile environment allowed us to extend the timeline strategically when high-value features (presentation mode) were identified

Collaboration & Cross-Functional Impact

Worked closely with data collectors team in Montevideo to understand metric collection workflows and ensure seamless integration between data input and user-facing display. Collaborated with business stakeholders through regular meetings using Whimsical boards for flowcharts and user stories. Maintained constant communication with the 10-member development team through Figma, ensuring design specifications were technically feasible and aligned with backend capabilities.

Design System Contribution

Created reusable components for data visualization, metric comparison panels, and excellence indicators that became part of The Hackett Group's design system. These components were later used across other platform features, demonstrating the value of systematic design thinking.
